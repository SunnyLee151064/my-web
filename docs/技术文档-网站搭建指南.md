### （一）一个网站包含什么？页面是怎么用代码搞出来的？

想象一下你要建一座房子。

**HTML = 房子的骨架**
```html
<!DOCTYPE html>
<html>
  <head>
    <title>我的个人博客</title>
  </head>
  <body>
    <header>欢迎来到我的博客</header>
    <main>
      <article>第一篇文章</article>
      <article>第二篇文章</article>
    </main>
  </body>
</html>
```
这就是 HTML——告诉浏览器哪里是墙（header），哪里是房间（main），哪里放东西（article）。

**CSS = 房子的装修**
```css
body {
  background-color: #f5f5f5;
  font-family: "微软雅黑", sans-serif;
}
header {
  background: linear-gradient(to right, #ff6b6b, #ffa500);
  padding: 20px;
}
```
没有 CSS 就是毛坯房，加上 CSS 才是漂亮的房子。

**JavaScript = 会动的家具**
```javascript
// 点击按钮，弹出一个对话框
document.getElementById('btn').addEventListener('click', () => {
  alert('你好啊，访问者！');
});
```
没有 JavaScript，网站就是静态的画。加上 JS 之后，按钮会动、菜单会展开、页面会刷新。

**我的网站用什么技术？**

我的网站用的是 **Next.js**，这是一个"高级装修队"。它把 HTML + CSS + JavaScript 打包在一起，让开发更简单。它用的是 React 框架，简单理解就是"组件化盖房子"——先把窗户做好，再把门做好，最后拼在一起就是一套房。

### （三）部署之后怎么访问？域名是什么？

**网址是怎么工作的？**

当你在浏览器输入 `my-web-orcin-phi.vercel.app` 时：
```
浏览器 → DNS服务器 → 找到对应的服务器 → 返回网站内容
```

- **DNS** = 网址的电话簿，把 `my-web-orcin-phi.vercel.app` 转换成服务器的 IP 地址
- **IP 地址** = 服务器的门牌号，比如 76.76.21.21

**我的域名是什么？**

Vercel 分配的免费域名是：`my-web-orcin-phi.vercel.app`

如果你买了自己的域名（比如 `sunny.ai`），可以设置 CNAME 记录指向 Vercel，这样更短更好记。

**访问流程图**
```
用户输入网址
    ↓
浏览器找 DNS 问路
    ↓
DNS 说：去找 Vercel
    ↓
Vercel 把网站内容发给你
    ↓
浏��器把内容渲染成页面
    ↓
你看到了网页！
```
